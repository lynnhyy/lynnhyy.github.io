Etude de données : le Nutriscore

  • Participants: Léo Ducruet, Moi
  • Logiciels: Shell Linux, PostgreSQL, RStudio
  • Temps : 4h par semaine sur 3 semaines

Le but du projet

PROBLEMATIQUE PRINCIPALE : Réalisation d’un guide d’installation rédigé en Anglais d'un serveur Debian 11 équipé d'Apache, PostgreSQL et PHP

Nous devions parcourir et trier une base de données fournie pour répondre à une problématique. L'objectif final était de produire plusieurs graphes sur RStudio permettant de répondre à la problématique et rédiger un rapport en anglais sur ce problème.

DONNEES ET SOURCES A DISPOSITION

Mon travail

    Rendus attendus :

  • - Une base de données nettoyée en fonction du sujet
  • - Un dossier gestion de projet
  • - Une liste de graphiques produits sur RStudio, extraits de la base de données afin de répondre au sujet

Première étape : Tri de la base de données

L’objectif de la première phase était de créer une nouvelle base de données à partir de la base de données fournie.

La base de données nous fournissait les données sur les produits alimentaires vendus avec un nutriscore. Nous devions retrouver les produits français transformés et supprimer les valeurs incorrectes.

Nous avons utilisé PostgreSQL pour explorer et trier la base de données afin de répondre au sujet. Nous ne voulions garder que le nécessaire pour répondre à notre problématique et supprimer les valeurs incorrectes (par exemple des valeurs négatives pour un code barre). Nous avions une liste de conditions à vérifier.

A la fin nous avions créé une nouvelle base triée avec uniquement des valeurs valides et les produits pouvant nous intéresser.

Ma part du travail :

Pour couvrir toutes les conditions, nous nous sommes réparti les commandes à réaliser sur SQL, et les avons toutes vérifiées individuellement. Nous avons donc tous deux participé activement à cette mission.

Deuxième étape : Réalisation des graphes

Pour cette phase, on nous a fourni la base de donnée triée que nous étions censé obtenir à la fin de la première phase afin d’éviter les problèmes concernant les valeurs incorrectes. Nous avons tout de même comparé avec la base de données que nous avions obtenue lors de la première phase et celle-ci était similaire à un détail près.

L’objectif de la seconde et dernière phase du projet était de fournir un bilan en anglais avec des graphes sur le calcul du nutriscore sur notre type de produit. Nous avons au final réussi à réaliser 5 graphes montrant les nutriscores selon les composants de nos produits. Le bilan en anglais et les graphes ont été réaliser sur Rstudio.

Ma part du travail :

Sur cette phase nous avons tous deux réalisé les graphes ensemble sur le même écran. Pour ce qui est de la rédaction, on s’est réparti équitablement le travail à faire puis nous avons tous les deux relu pour vérifier qu’il n’y avait pas de faute dans le document.
